# Demo Walkthrough

Use this 5-step flow when presenting the project in an interview.

## 1. Start With The Real Workflow Problem

Explain that retention agents often spend several minutes checking why a customer's bill increased. The risk is quoting the wrong price because discounts, add-ons, device financing, roaming, or one-time charges are easy to miss.

## 2. Open The Static Demo

Open the GitHub Pages demo and point out that the public version uses synthetic data only. This protects customer privacy while still showing the product workflow.

## 3. Walk Through The Bill Breakdown

Show the summary cards and service table. Explain how the UI separates:

- original plan price;
- visible discounts and expiry dates;
- current monthly amount before tax;
- this-bill total;
- bill drivers such as roaming, one-time charges, device payment, and promo expiry.

## 4. Click A Line With HUP / Financing

Click the demo HUP wireless line. Show that device financing details stay collapsed until the agent needs them. Explain why this matters: most calls need a quick scan first, then deeper details only when relevant.

## 5. Show Agent Assist And Note Output

Type or edit a customer statement in the transcript box. Show how the rule-based Agent Assist updates the call stage, suggested response, follow-up question, and Maestro note preview. End by explaining that a production version would connect to official systems before quoting final offers.
